● Ótima pergunta — esse é um desafio clássico de SaaS multi-tenant com
  rastreamento. Conheço bem essa abordagem. Vou explicar a arquitetura antes de
  implementar.

  ---
  O problema central

  Você não pode botar o pixel direto no JS do frontend porque cada empresa tem
  um pixel diferente. Mas existem duas camadas onde isso funciona:

  ---
  Arquitetura: duas abordagens que se complementam

  1. Server-Side via API (funciona 100% no seu caso)

  Quando o WhatsApp recebe a primeira mensagem de um número novo, o backend
  dispara o evento diretamente para a Meta/Google via API server-side:

  - Meta → Conversions API (CAPI): POST 
  https://graph.facebook.com/v18.0/{pixel_id}/events
  - Google → Offline Conversions API: POST https://googleads.googleapis.com/...

  Cada empresa configura no painel: pixel_id + access_token (Meta) e
  conversion_id + conversion_label (Google).

  Vantagem: funciona para qualquer conversa WhatsApp, independente de como o
  cliente chegou. Não precisa de nada no frontend.

  ---
  2. Client-Side (para landing pages / widget)

  Se o cliente vier de um link tipo seuapp.com/c/empresa-slug, a página carrega
  dinamicamente o pixel daquela empresa e dispara na abertura.

  ---
  O que implementar no sistema

  Empresa
    meta_pixel_id        → "123456789"
    meta_capi_token      → "EAAxxxxxxx..." (token da API)
    google_ads_id        → "AW-123456789"
    google_ads_label     → "AbCd..." (label da conversão)
    tracking_ativo       → bool

  Fluxo quando nova conversa chega:

  WhatsApp msg recebida
    → Celery task: processar_mensagem_recebida
      → se primeira mensagem deste número:
        → _disparar_conversao_meta(empresa, numero, evento="Lead")
        → _disparar_conversao_google(empresa, numero, evento="contact")

  ---
  Quer que eu implemente isso agora? O que teria:

  1. Migration SQL com campos de tracking na tabela empresa
  2. Serviço app/services/tracking.py com funções Meta CAPI + Google Ads
  3. Hook na task de nova conversa para disparar o evento
  4. Campos de configuração na API de empresa (PUT /empresa/configuracoes)
  5. Seção "Rastreamento" no painel React com campos para colar os IDs

  Para Meta CAPI você precisa do Pixel ID + Access Token (gerado no Events
  Manager). Para Google Ads precisa do Conversion ID (AW-XXXXXXXXX) + Conversion
   Label. Você ou a empresa já tem esses dados disponíveis?

🚀 GUIA RÁPIDO DE INSTALAÇÃO

1. Instalar dependência:
   $ cd backend_fastapi
   $ pip install prometheus-client==0.19.0

2. Restart serviços:
   $ docker-compose restart api celery_worker

3. Verificar logs:
   $ docker-compose logs -f api | grep "Redis Pub/Sub"
   # Deve aparecer: ✅ Redis Pub/Sub conectado

4. Testar métricas:
   $ curl http://localhost:8000/metrics

5. Monitorar Redis:
   $ docker exec -it whatsapp_redis redis-cli PSUBSCRIBE "ws:broadcast:emp:*"

🧪 TESTES ESSENCIAIS

[✓] Redis Pub/Sub funcionando
    → Monitorar canal: ws:broadcast:emp:*
    → Enviar mensagem e verificar broadcast

[✓] Deduplicação backend
    → Verificar chaves: msg:processed:*
    → Enviar msg duplicada e confirmar skip

[✓] Deduplicação frontend
    → Console deve mostrar: "Mensagem duplicada (ignorada)"

[✓] Métricas Prometheus
    → GET /metrics deve retornar texto/plain
    → Verificar: websocket_connections_total

[✓] Circuit breaker
    → Estado inicial: circuit_breaker_state = 0.0 (CLOSED)

📈 MÉTRICAS DISPONÍVEIS (Prometheus)

Conexões:
  • websocket_connections_total{empresa_id}
  • websocket_active_connections{empresa_id}
  • websocket_total_active_connections

Latências:
  • pubsub_latency_seconds (histogram)
  • broadcast_latency_seconds (histogram)
  • whatsapp_api_latency_seconds (histogram)

Mensagens:
  • whatsapp_sent_total{status}
  • whatsapp_received_total{empresa_id}
  • ws_broadcasts_total{event,status}

Cache:
  • cache_hits_total{cache_type}
  • cache_misses_total{cache_type}
  • cache_hit_rate{cache_type}

Sistema:
  • circuit_breaker_state{component}  (0=CLOSED, 1=HALF_OPEN, 2=OPEN)
  • circuit_breaker_failures{component}
  • redis_pubsub_connected  (0=down, 1=up)
